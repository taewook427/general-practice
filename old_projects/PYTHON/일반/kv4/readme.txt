1. 개발 목적, 용도
KVault4는 소프트웨어적으로 구현된 암호화 볼트로, 중규모 저장소로의 사용을 목적하고 있습니다.
로컬 컴퓨터에서 실행되는 KVault4를 응용하여 KCloud4 또한 제작될 예정입니다.
권장 파일 개수 한도는 2천5백만 개이며, 이때 램 사용량은 약 8GB 정도 입니다.
파일 개수 5천개 이하의 일상적인 소규모 저장소로 사용할 때에는 약 2~3GB의 램을 사용합니다.

2. 용어
물리 저장소 : HDD/SSD/USB 등 실제 파일/폴더가 저장되고 운영체제에서 접근할 수 있는 저장소.
클러스터 : 물리 저장소 상에 위치하며, 컨테이너와 헤더 파일을 가지고 있는 폴더.
컨테이너 : 물리 저장소 상에 위치하며, 암호화된 kv4 파일을 담고 있는 폴더.
헤더 : 물리 저장소 상에 위치하며, 파일 구조와 키가 암호화 되어있는 파일.
스트럭트 : KVault4에서 폴더와 비슷한 기능을 하는 단위.
아톰 : KVault4에서 파일과 비슷한 기능을 하는 단위.
깊이 : 상위 스트럭트의 개수. 부팅 시 보이는 특별 스트럭트 root는 -1, bin/은 0, main/도 0.

3. 로그인
처음 프로그램 시작 시 로그인 화면이 보입니다.
먼저 클러스터를 선택해야 합니다. CLU 버튼으로 클러스터 폴더를 선택하십시오.
빈 폴더인 경우 새 클러스터가 생성됩니다.
올바른 클러스터를 선택한 경우 암호 힌트가 표시됩니다.
키 파일 경로와 비밀번호를 올바르게 입력하고 GO 버튼을 누르면 클러스터가 부팅됩니다.

4. Import
이 메뉴는 파일을 암호화하여 가져옵니다. 깊이 0 이상의 스트럭트에서만 동작합니다.
Files : 파일들을 선택하고 가져옵니다. 파일들은 현재 보이는 스트럭트 안에 들어갑니다.
Folder : 폴더를 선택하고 가져옵니다. 폴더가 덩어리화된 후에 파일처럼 암호화됩니다.
즉, 내보낼 때는 정상적으로 폴더로 풀리나, KVault4 안에서는 파일처럼 행동합니다.
Full Folder : 폴더를 선택하고 가져옵니다. 폴더의 하위 구조와 파일이 모두 동일하게 들어옵니다.
즉, 가져올 폴더가 하위 폴더와 파일이 있다면, 하위 스트럭트와 아톰이 존재하는 스트럭트가 생성됩니다.

5. Export
이 메뉴는 파일을 복호화합니다. 단, 저장소 내 데이터를 삭제하지 않으며, 영향도 끼치지 않습니다.
Atoms : 깊이 0 이상의 스트럭트에서만 동작하며, 선택한 아톰들을 지정된 폴더에 모두 복호화합니다.
Struct : 선택한 스트럭트를 하위 구조 그대로 복호화합니다.
즉, 하위 스트럭트와 아톰이 있는 스트럭트라면 하위 폴더와 파일이 있는 폴더가 생성됩니다.

6. Files
이 메뉴는 저장소 파일 관련 동작을 진행합니다. 깊이 0 이상의 스트럭트에서만 동작합니다.
New Struct : 현재 스트럭트 안에 새로운 빈 스트럭트를 생성합니다.
Move : 선택한 스트럭트 혹은 아톰을 새로운 스트럭트 안으로 이동시킵니다.
Recycle : 선택한 스트럭트 혹은 아톰을 특수 스트럭트 bin/으로 이동시킵니다.

7. Action
이 메뉴는 저장소 파일 관련 특수 동작을 진행합니다. 깊이 0 이상의 스트럭트에서만 동작합니다.
Rename : 선택한 스트럭트 혹은 아톰의 이름을 바꿉니다.
Sort Name : 현재 스트럭트 내부 항목들의 순서를 이름순으로 정렬합니다.
정렬하지 않은 스트럭트는 최근에 가져온 항목이 위에 놓여 있습니다.
Terminate : 특수 스트럭트 bin/에서만 동작합니다.
선택한 bin/의 스트럭트 혹은 아톰을 저장소에서 영구적으로 삭제합니다.

8. Manage
이 메뉴는 저장소 관리 기능을 수행합니다.
Info : 선택한 스트럭트 혹은 아톰에 대한 정보를 팝업으로 띄웁니다.
Reset PW : 비밀번호, 키 파일 경로, 힌트를 재설정합니다.
Debug : 마지막 명령 정보와 디버그 정보를 제공합니다.
조작 실패 후 에러 메세지가 표시되면, 다른 조작을 하지 말고 디버그 정보를 확인하여 문제를 파악하십시오.
Clean : 물리 저장소와 KVault4 저장소 데이터를 비교하여 동일성을 유지시킵니다.
각 데이터 항목 별로 사라진 파일에 대한 정보가 표시됩니다. 세 항목 모두 빈 리스트여야 정상입니다.
조작에 실패한 경우, 디버그 정보 확인 후 종료-재부팅하여 clean 명령을 실행하여 데이터 꼬임을 방지하는 것이 권장됩니다.

9. Others
이 메뉴는 기타 명령을 실행합니다.
Settings : 설정 텍스트 파일을 실행합니다. 변화된 사항은 다음 부팅부터 적용됩니다.
Help : 설명서 파일을 실행합니다.

10. 조작
더블클릭하여 하위 스트럭트로 이동할 수 있습니다.
리스트박스 위에서 드래그하여 여러 항목을 선택할 수 있습니다.
단, 단일 항목 대상 명령인 경우 선택된 항목들 중 가장 위의 항목만을 대상으로 실행됩니다.
< 버튼으로 상위 스트럭트로 이동합니다.
? 버튼으로 검색을 진행합니다. 검색은 검색창 위쪽에 보이는 스트럭트의 내부 항목을 대상으로 합니다.
* 버튼은 확인 버튼입니다. 명령을 실행시킬 때 사용합니다.
